# Концепция проекта ARIA

## Интеллектуальная информационно-справочная платформа ИКТИБ ЮФУ

---

## 1. Общие сведения

**Наименование:** ARIA - AI Reference & Information Assistant

**Тип:** Междисциплинарный проект

**Назначение:** Интеллектуальная справочная платформа для студентов и абитуриентов ИКТИБ ЮФУ, обеспечивающая получение достоверной информации об учебном процессе и инфраструктуре института через Telegram-бот и веб-приложение с применением генеративного ИИ.

### Название проекта

Название ARIA выбрано осознанно и несёт несколько уровней смысла:

- **Бэкроним:** AI Reference & Information Assistant - буквально отражает функцию системы.
- **Персонификация:** ARIA - это не безликий инструмент, а собеседник, к которому можно обратиться с вопросом. Исследования в области Human-Computer Interaction показывают, что персонифицированные интерфейсы повышают вовлечённость пользователей и снижают порог входа.
- **Отсылка к WAI-ARIA** (Accessible Rich Internet Applications) - веб-спецификация, предоставляющая дополнительный контекст и информацию там, где стандартные средства недостаточны. Проект ARIA решает аналогичную задачу: предоставляет студентам дополнительный контекст и доступ к информации, рассредоточенной по множеству источников.

---

## 2. Актуальность

Студенты и абитуриенты ИКТИБ ежедневно нуждаются в информации по широкому кругу вопросов: поступление, учебные программы, сессии, справки, стипендии, общежития, контакты подразделений. Эта информация рассредоточена по множеству источников - сайт, соцсети, деканат - что порождает ряд проблем:

- **Отсутствие единой точки входа** - студенты тратят значительное время на поиск ответов на типовые вопросы.
- **Нагрузка на персонал** - сотрудники деканата многократно отвечают на однотипные обращения.
- **Временные ограничения** - доступ к информации ограничен рабочими часами подразделений.

### Отличие от существующих систем

ARIA не дублирует функции существующих систем. LMS - это управление курсами и оценками. Modeus - расписание. ARIA - справочный ассистент, который отвечает на вопросы, не покрываемые ни одной из этих систем: порядок получения справок, условия назначения стипендий, правила заселения в общежитие, информация о пересдачах, внеучебная деятельность и десятки других тем.

Более того, ARIA выступает **агрегатором**: платформа объединяет статичную базу знаний с динамическими данными из внешних источников (расписание, объявления, новости), предоставляя единый интерфейс на естественном языке.

**Решение:** единая платформа, объединяющая верифицированную базу знаний института с технологией Retrieval-Augmented Generation (RAG). В отличие от обычных чат-ботов, RAG позволяет генерировать ответы с опорой на конкретные документы института, а не на общие данные языковой модели, что обеспечивает точность и актуальность ответов. Платформа доступна через Telegram - наиболее распространённый мессенджер среди студентов.

---

## 3. Цели и задачи

**Цель:** разработка интеллектуальной платформы для автоматизированного обслуживания информационных запросов студентов и абитуриентов ИКТИБ ЮФУ.

**Задачи:**

1. Проектирование микросервисной архитектуры с независимым развёртыванием компонентов.
2. Разработка AI-сервиса с конвейером обработки: классификация запроса, семантический поиск по базе знаний (RAG), генерация ответа через LLM, ведение контекста диалога.
3. Формирование верифицированной базы знаний по ключевым тематикам института.
4. Реализация Telegram-бота и Telegram Mini App как пользовательских интерфейсов.
5. Создание административной панели для управления контентом и аналитики.
6. Интеграция с внешними источниками данных (расписание, объявления) для ответов на динамические вопросы.
7. Обеспечение безопасности: JWT-аутентификация, ролевая модель доступа, rate limiting.
8. Контейнеризация и автоматизация развёртывания через Docker Compose.

---

## 4. Архитектура

Система построена по микросервисному принципу с паттерном API Gateway. Сервисы взаимодействуют между собой по TCP, с внешними клиентами - по HTTP/REST. Кодовая база организована как монорепозиторий (npm workspaces).

### Компоненты

- **API Gateway** - единая точка входа для HTTP-запросов, маршрутизация, аутентификация и авторизация.
- **AI-сервис** - ядро платформы: классификация вопросов, векторный поиск по базе знаний (RAG), обращение к LLM, семантическое кеширование, ведение истории диалогов.
- **User-сервис** - управление пользователями, ролевая модель (студент/модератор/администратор), система блокировки, контроль лимитов.
- **Bot-сервис** - интерфейс Telegram-бота: команды, обработка сообщений, категории вопросов, оценка ответов.
- **Data-сервис** *(планируется)* - агрегация динамических данных из внешних источников (расписание, объявления, новости) посредством парсинга с кешированием.
- **Административная панель** - веб-приложение для управления базой знаний, пользователями, просмотра аналитики и статистики.
- **Telegram Mini App** - веб-приложение внутри Telegram с расширенным интерфейсом: чат с потоковым выводом, профиль, история обращений, настройки.

### Хранилища данных

- **PostgreSQL** с расширением **pgvector** - реляционное хранилище и векторный поиск для RAG и семантического кеша.
- **Redis** - оперативное хранилище: лимиты пользователей, cooldown, кеширование.

### Принцип работы RAG-конвейера

Пользователь задаёт вопрос -> система классифицирует его и преобразует в вектор -> ищет похожие вопросы в семантическом кеше -> при промахе находит релевантные фрагменты базы знаний по векторному сходству -> формирует контекст (системный промпт + фрагменты + история диалога) -> отправляет в LLM -> возвращает ответ пользователю, сохраняя его в кеш для ускорения будущих запросов.

## 5. Стек технологий

**Backend:** Node.js, NestJS, TypeScript, TypeORM, Passport.js (JWT), Telegraf.js, OpenAI SDK, Xenova/Transformers (локальные embeddings)

**Frontend:** Next.js, React 19, TypeScript, Tailwind CSS, Recharts, Telegram WebApp SDK

**Инфраструктура:** Docker Compose, PostgreSQL 17 + pgvector, Redis 7.2, Swagger/OpenAPI

## 6. База знаний

Информационное наполнение охватывает ключевые направления: поступление, образовательные программы, учебный процесс и сессии, стипендии, общежития, справки и документы, внеучебная деятельность, контакты ИКТИБ. Материалы хранятся в структурированных markdown-файлах, управляемых через административную панель.

При индексации файлы разбиваются на семантические фрагменты, для каждого вычисляется векторное вложение. При поступлении вопроса система находит наиболее релевантные фрагменты и включает их в контекст запроса к LLM, что обеспечивает фактологическую точность ответов.

Помимо статичной базы знаний, планируется интеграция динамических данных - расписание занятий, объявления и новости с сайта ИКТИБ - посредством периодического парсинга и кеширования. Это позволит ARIA отвечать не только на общие справочные вопросы, но и на актуальные: "что у моей группы завтра?", "какие новости на сайте института?".

## 7. Безопасность

- **Два контура аутентификации:** пользователи - через Telegram ID + JWT, администраторы - через логин/пароль (bcrypt) + JWT.
- **Ролевая модель (RBAC):** студент, модератор, администратор - с разграничением доступа на уровне API Gateway.
- **Rate limiting:** ежедневные лимиты токенов и запросов на пользователя, cooldown между обращениями.
- **Валидация** входных данных на всех уровнях.

---

## 8. Ожидаемые результаты

1. **Снижение нагрузки на персонал** - автоматизация ответов на типовые вопросы.
2. **Круглосуточная доступность** справочной информации.
3. **Точность ответов** за счёт RAG: генерация на основе верифицированных данных института.
4. **Агрегация источников** - единый интерфейс для информации, рассредоточенной по сайтам, документам и системам.
5. **Аналитика обращений** - выявление востребованных тематик для обновления контента.
6. **Масштабируемость** - архитектура допускает адаптацию для других подразделений ЮФУ.
